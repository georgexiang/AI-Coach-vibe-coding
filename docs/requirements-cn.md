# AI 教练解决方案 —— 需求规格说明书

> 摘自凯捷 AI 教练 AWS 解决方案文档

---

## FR：功能性需求

### FR-1：培训材料管理

| ID | 需求描述 | 优先级 |
|----|----------|--------|
| FR-1.1 | 集中化文档管理，支持 Word / Excel / PDF / 内容上传 | 高 |
| FR-1.2 | 培训材料的版本控制与归档 | 高 |
| FR-1.3 | 按可配置的保留策略自动删除语音记录 | 中 |
| FR-1.4 | 离职员工历史数据归档 | 中 |

### FR-2：一对一 HCP 拜访面谈

| ID | 需求描述 | 优先级 |
|----|----------|--------|
| FR-2.1 | 模拟包含异议处理的面对面拜访场景 | 高 |
| FR-2.2 | 在交互前向医药代表展示场景背景及数字化 HCP 角色信息 | 高 |
| FR-2.3 | 接受医药代表的语音输入（通过 ASR 语音识别进行交互） | 高 |
| FR-2.4 | 接受医药代表的文字输入（文字交互方式） | 高 |
| FR-2.5 | 生成数字化 HCP 对应的语音/文字输出 | 高 |
| FR-2.6 | 支持历史对话回放与复盘（往期会话回放） | 中 |
| FR-2.7 | 基于可配置评分标准提供评分与反馈 | 高 |
| FR-2.8 | 提供可自定义的评分标准与反馈体系 | 高 |

### FR-3：虚拟科室学术会议演讲（一对多）

| ID | 需求描述 | 优先级 |
|----|----------|--------|
| FR-3.1 | 支持包含产品/疾病知识的内容演讲 | 高 |
| FR-3.2 | 支持虚拟 HCP 的听众提问回答 | 高 |
| FR-3.3 | 向医药代表提供演讲多维场景信息 | 中 |
| FR-3.4 | 支持语音输入并在屏幕上实时转写文字（实时字幕） | 高 |
| FR-3.5 | 基于历史数据生成典型异议 | 中 |
| FR-3.6 | 提供处理异议的言语建议 | 中 |
| FR-3.7 | 提供可自定义的评分标准与反馈体系 | 高 |

### FR-4：多维度评分系统

| ID | 需求描述 | 优先级 |
|----|----------|--------|
| FR-4.1 | 对关键能力进行多维度综合评分 | 高 |
| FR-4.2 | 在培训过程中提供实时改进建议 | 高 |
| FR-4.3 | 课后报告呈现优势分析 | 高 |
| FR-4.4 | 课后报告呈现不足分析 | 高 |
| FR-4.5 | 课后报告提供针对性改进方向 | 高 |
| FR-4.6 | 覆盖以下维度的评分：核心信息传递、异议处理、沟通能力、产品知识、科学信息表达 | 高 |

### FR-5：报告与数据看板

| ID | 需求描述 | 优先级 |
|----|----------|--------|
| FR-5.1 | 生成按业务单元（BU）、职级、时间段可排序/筛选的标准报告 | 高 |
| FR-5.2 | 提供个人维度的培训结果分析 | 高 |
| FR-5.3 | 提供团队维度的培训结果分析 | 高 |
| FR-5.4 | 支持 PDF 格式导出 | 中 |
| FR-5.5 | 支持 Excel 格式导出 | 中 |
| FR-5.6 | 跟踪全组织的培训进度与完成状态 | 高 |

### FR-6：数字化 HCP 配置

| ID | 需求描述 | 优先级 |
|----|----------|--------|
| FR-6.1 | 设置虚拟 HCP 的形象/视觉外观 | 中 |
| FR-6.2 | 设置 HCP 的知识背景与医学观点 | 高 |
| FR-6.3 | 按场景配置评分标准 | 高 |
| FR-6.4 | 为虚拟 HCP 配置多样化的人格设定 | 中 |
| FR-6.5 | 为 HCP 配置不同的情绪状态、情境和意图 | 中 |
| FR-6.6 | 支持多产品培训场景 | 高 |

### FR-7：AI 与 NLP 能力

| ID | 需求描述 | 优先级 |
|----|----------|--------|
| FR-7.1 | 拟人化情感深度的对话体验（文本分析驱动） | 高 |
| FR-7.2 | 语音处理 / ASR（自动语音识别） | 高 |
| FR-7.3 | NLP 分析复杂表达：情绪、语境、意图 | 高 |
| FR-7.4 | 充分发挥 AI 能力的动态课程优化 | 中 |
| FR-7.5 | 基于医药代表职级/业务单元的个性化培训路径 | 高 |

## NFR：非功能性需求

| ID | 需求描述 | 类别 | 优先级 |
|----|----------|------|--------|
| NFR-1 | 满足合规审计的能力跟踪（全程可追溯的培训路径） | 合规性 | 高 |
| NFR-2 | 可扩展，支持全组织规模部署 | 可扩展性 | 高 |
| NFR-3 | 可针对特定治疗领域、新产品上市或区域规范灵活配置场景 | 可扩展性 | 中 |
| NFR-4 | 可适应新兴市场趋势或法规变化 | 可维护性 | 中 |
| NFR-5 | 符合数据保留政策合规要求（自动删除语音记录） | 数据隐私 | 高 |
| NFR-6 | 离职员工历史数据归档 | 数据管理 | 中 |
| NFR-7 | AWS 云端部署 | 基础设施 | 高 |
