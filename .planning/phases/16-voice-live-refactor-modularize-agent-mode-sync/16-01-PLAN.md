**1. Create `frontend/src/lib/voice-utils.ts`:**

```typescript
/**
 * Encode Float32Array PCM audio to base64-encoded Int16 PCM for Azure Voice Live.
 * Clips values to [-1, 1] range to prevent audio artifacts.
 */
export function encodePcmToBase64(audioData: Float32Array): string {
  const int16 = new Int16Array(audioData.length);
  for (let i = 0; i < audioData.length; i++) {
    const s = Math.max(-1, Math.min(1, audioData[i] ?? 0));
    int16[i] = s < 0 ? s * 0x8000 : s * 0x7fff;
  }
  const bytes = new Uint8Array(int16.buffer);
  let binary = "";
  for (let i = 0; i < bytes.length; i++) {
    binary += String.fromCharCode(bytes[i]!);
  }
  return btoa(binary);
}
```

- **Reentrancy guard** (per owner decision RD-6): Use a `useRef<boolean>(false)` called `busyRef`. If `busyRef.current === true` when `startSession` is called, return immediately (no-op). Set it to `true` at start, `false` in finally block.

The `startSession` function sequence:
1. Check busyRef reentrancy guard — if already busy, return immediately
2. Create new AbortController, store in abortRef
3. audioHandler.initialize() — with mic error detection (NotAllowedError)
4. Check abortRef.current?.signal.aborted — bail if true
5. Wire avatarSdpCallbackRef
6. voiceLive.connect(hcpProfileId, systemPrompt, vlInstanceId) — check aborted after
7. Call onModeResolved(result.avatarEnabled)
8. If avatarEnabled, avatarStream.connect(...) — check aborted after
9. audioHandler.startRecording with `encodePcmToBase64` from voice-utils
10. Set busyRef = false in finally

The `stopSession` function:
1. abortRef.current?.abort() — cancel any in-flight connection
2. voiceLive.disconnect()
3. avatarStream.disconnect()
4. audioHandler.cleanup()
5. audioPlayer.stopAudio()

Expose `isBusy` as a reactive value via `useState<boolean>(false)` that mirrors busyRef state.
