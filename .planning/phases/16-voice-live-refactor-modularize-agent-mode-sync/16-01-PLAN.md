# Plan 16-01: Frontend Voice Live Modularization

**Phase:** 16 ‚Äî Voice Live Refactor
**Requirement:** VL-16-01
**Depends on:** None
**Autonomous:** true

## Goal

Extract duplicated Voice Live logic into shared modules. The platform's voice capability is the core differentiator ‚Äî it must be modular, consistent, and maintainable.

## Tasks

### Task 1: Extract voice utility functions

**Files to create/modify:**
- CREATE `frontend/src/lib/voice-utils.ts` ‚Äî shared voice utilities

**Extract from `voice-session.tsx` + `voice-test-playground.tsx`:**
1. `encodePcmToBase64(audioData: Float32Array): string` ‚Äî the PCM Float32‚ÜíInt16‚Üíbase64 encoding (currently ~12 lines duplicated in both files)
2. `createVoiceSessionConfig(params): SessionUpdateMessage` ‚Äî build the session.update message structure

**Update both consumers** to import from `@/lib/voice-utils`.

### Task 2: Extract shared voice initialization hook

**Files to create/modify:**
- CREATE `frontend/src/hooks/use-voice-session-lifecycle.ts`

Extract the ~50 lines of shared init sequence from `voice-session.tsx` (initVoice) and `voice-test-playground.tsx` (startTest):
1. Initialize audio handler (mic permissions)
2. Wire avatar SDP callback
3. Call voiceLive.connect()
4. Connect avatar stream (if avatar enabled)
5. Start recording with PCM encoding via `encodePcmToBase64`

The hook should accept: `{ voiceLive, audioHandler, avatarStream, onTranscript }` and return `{ startSession, stopSession }`.

### Task 3: Extract shared AssignHcpDialog component

**Files to create/modify:**
- CREATE `frontend/src/components/admin/assign-hcp-dialog.tsx`

Extract the assign-to-HCP dialog from 3 locations:
- `voice-live-management.tsx` lines 106-158
- `vl-instance-editor.tsx` lines 324-340
- `agent-config-left-panel.tsx` lines 75-118

Shared component props: `{ instanceId, open, onOpenChange, onAssigned }`. Uses `useHcpProfiles()` internally, filters already-assigned profiles.

Update all 3 consumers to use `<AssignHcpDialog>`.

### Task 4: Extract shared voice constants

**Files to create/modify:**

Move from `vl-instance-dialog.tsx` and `vl-instance-editor.tsx`:

- `TURN_DETECTION_TYPES`
- `RECOGNITION_LANGUAGES`
- `DEFAULT_VL_INSTANCE_FORM` (complete version with all fields)

Update both consumers + any other files that define these locally.
