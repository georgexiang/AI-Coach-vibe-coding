### Task 2: WebSocket proxy dual-mode connect

**File:** `backend/app/services/voice_live_websocket.py`

**Changes:**
1. Remove outdated comments ("Agent mode requires Azure AD/Entra ID auth")
2. Update `_load_connection_config()` to return `agent_mode: bool`, `agent_name: str`, `project_name: str` when HCP has synced agent
3. In `handle_voice_live_websocket()`, replace the single `connect(model=...)` with dual-mode:

```python
from azure.ai.voicelive.aio import AgentSessionConfig, connect

if use_agent_mode and agent_name and project_name:
    agent_config: AgentSessionConfig = {
        "agent_name": agent_name,
        "project_name": project_name,
    }
    connection = connect(
        endpoint=endpoint,
        credential=AzureKeyCredential(api_key),
        agent_config=agent_config,
    )
else:
    connection = connect(
        endpoint=endpoint,
        credential=AzureKeyCredential(api_key),
        model=model,
    )
```

4. Agent mode session.update: only send `modalities` (Agent carries own instructions). Model mode: send full config as before.
5. Add fallback: if Agent connect fails, retry in Model mode with warning log + `proxy.mode_degraded` event to frontend.
6. Remove API version hardcode (`2025-05-01-preview`) — let SDK use its default (`2026-01-01-preview`).
