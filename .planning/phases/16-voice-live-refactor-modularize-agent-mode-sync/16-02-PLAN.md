Purpose: Enable HCP pages to use Agent mode (AI Foundry pre-configured instructions) while VL Instance editor uses Model mode (custom instructions). The two modes have fundamentally different behaviors — silent degradation would mislead users.

Output: Updated WebSocket proxy with dual-mode connect (no fallback), upgraded SDK, token broker with agent_mode_available flag.

**CRITICAL per owner decision RD-2:** There is NO fallback from Agent mode to Model mode. If Agent mode `connect()` raises an exception, it propagates to the outer try/except which sends the error to the frontend via WebSocket error message. The user sees the error. Do NOT add any `except` block that retries in Model mode. Do NOT add any fallback logic, fallback chain, degraded mode, or silent retry. The two modes have completely different behaviors — silent degradation would mislead the user.

e) Remove the `api_version=AZURE_VOICE_API_VERSION` parameter from the `connect()` calls — let the SDK use its default.
