## Goal

Fix the gap where HCP profiles bound to VoiceLiveInstances don't sync voice/avatar config to AI Foundry agents. After this plan, opening an Agent in AI Foundry portal shows correct voice configuration.

## Tasks

### Task 1: Fix build_voice_live_metadata to use resolve_voice_config

**File:** `backend/app/services/agent_sync_service.py`

Current problem: `build_voice_live_metadata(profile)` reads deprecated inline HcpProfile fields via `getattr(profile, "voice_name", ...)`. It never calls `resolve_voice_config()` which prefers VoiceLiveInstance settings.

**Changes:**
1. Import `resolve_voice_config` from `voice_live_instance_service`
2. In `build_voice_live_metadata()`, call `resolve_voice_config(profile)` to get the effective config dict
3. Read all voice fields from the resolved config instead of raw profile attributes
4. Add avatar fields to metadata:
   - `avatar.character`, `avatar.style`, `avatar.customized`, `avatar.enabled`
5. Add VoiceLiveInstance-only fields:
   - `response_temperature`, `proactive_engagement`, `auto_detect_language`
   - `playback_speed`, `custom_lexicon_enabled`, `custom_lexicon_url`
6. Keep the 512-char chunking for Azure metadata limits

### Task 2: Trigger re-sync when VoiceLiveInstance is updated

**File:** `backend/app/services/voice_live_instance_service.py`

When a VoiceLiveInstance is updated (`update_instance()`):
1. After saving, find all HcpProfiles with `voice_live_instance_id == instance.id`
2. For each profile with `agent_sync_status == "synced"` and valid `agent_id`:
   - Call `sync_agent_for_profile(profile, sync_config)` to push updated config
3. Log count of re-synced profiles

When a VoiceLiveInstance is assigned/unassigned:
1. Re-sync the affected HCP profile's agent (if it has one)

### Task 3: Trigger re-sync when VoiceLiveInstance is assigned to HCP

**File:** `backend/app/services/voice_live_instance_service.py`

In `assign_to_hcp()` and `unassign_from_hcp()`:
1. After DB save, if the HCP profile has a synced agent, trigger agent sync
2. This ensures portal reflects current VL binding state
