# Phase 8 -- UI Design Contract

> Visual and interaction contract for the Voice & Avatar Demo Integration phase: voice session mode selector, avatar video view with waveform fallback, voice controls, real-time transcript overlay, admin config extensions for Voice Live and Avatar, and three-mode graceful fallback chain (avatar+voice, voice-only, text-only). Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | manual shadcn/ui adaptation (no `components.json`) -- inherited from Phase 1 |
| Preset | not applicable -- components adapted from local Figma Make exports |
| Component library | Radix UI primitives via shadcn/ui wrappers |
| Icon library | lucide-react ^0.460.0 |
| Font | Inter (EN) + Noto Sans SC (CN), loaded via Google Fonts |

**Source:** Phase 1 UI-SPEC (established design system); no changes for Phase 8.

**Note:** Phase 8 introduces no new UI primitive dependencies from Radix/shadcn. All new voice and avatar components are compositions of existing primitives (Button, Dialog, ScrollArea, Switch, Tooltip, Card, Badge, Avatar, Skeleton) and project-specific components. The only new frontend dependency is `rt-client` v0.5.2 (Azure Voice Live SDK, installed from local tgz). No new Radix primitives or shadcn/ui base components are required.

## Spacing Scale

Declared values (inherited from Phase 1, multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, status indicator spacing |
| sm | 8px | Compact element spacing, mode selector button gaps |
| md | 16px | Default element spacing, panel internal padding, voice controls padding |
| lg | 24px | Section padding inside panels, avatar view internal padding |
| xl | 32px | Layout gaps between major panels |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Session header height (64px, matching existing coaching header) |

Exceptions:
- Avatar video area height (embedded mode): 280px (35 * 8px) -- taller than conference stage (200px) because avatar video is the primary visual element in voice sessions
- Avatar video area height (full-screen mode): `calc(100vh - 64px - 80px)` -- full viewport minus header (64px) and floating transcript bar (80px)
- Waveform visualization height: 120px (15 * 8px) -- fallback display area when avatar is unavailable
- Mode selector bar height: 48px (6 * 8px) -- pill-style segmented control for text/voice/avatar mode
- Floating transcript overlay height: 80px (10 * 8px) -- semi-transparent bar at bottom of full-screen avatar view
- Voice control bar height: 64px (8 * 8px) -- centered mic button with status indicators
- Mobile touch target minimum: 44px (WCAG 2.5.5 conformance, inherited)
- Mic button diameter: 56px (7 * 8px) -- larger than standard button for primary voice interaction affordance

**Source:** Phase 1 UI-SPEC spacing scale; CONTEXT.md D-05 (embedded vs full-screen); RESEARCH.md Pattern 2/3 (avatar/voice layout)

## Interaction States

### Voice Session Lifecycle States

| State | Visual | User Actions Available |
|-------|--------|----------------------|
| Mode Selection (pre-session) | Mode selector visible on scenario selection or session start. Text always available; Voice and Avatar show availability badges or disabled tooltips | Select mode, start session |
| Connecting | Connection status shows orange dot + "Connecting...", mic button disabled with Loader2 spinner, avatar area shows Skeleton placeholder, transcript empty | Wait, or cancel to fall back to text |
| Connected (idle) | Connection status shows green dot + "Connected", mic button pulses subtly to invite interaction, avatar shows idle animation or waveform shows flat line, "Ready" label | Speak into mic, or toggle to keyboard input |
| Listening (user speaking) | Mic button turns green with pulse animation, waveform shows active amplitude, transcript shows partial user text in real-time with typing cursor, avatar shows listening pose | Speak naturally; VAD detects end of speech automatically |
| Processing (AI responding) | Mic button turns orange, waveform shows AI speech amplitude, transcript shows streaming assistant text, avatar lip-syncs with audio response | Listen to response; can interrupt by speaking (barge-in via VAD) |
| Fallback: avatar failed | Toast notification "Avatar connection failed. Switching to voice-only mode.", avatar area transitions to waveform visualization (300ms fade), session continues | Continue voice interaction without avatar |
| Fallback: voice failed | Toast notification "Voice connection failed. Switching to text mode.", voice controls replaced by ChatInput, session continues as text-only | Type messages as in standard F2F session |
| Session ended | All controls disabled, "Session complete -- View Results" banner, connection auto-closed | Navigate to scoring page |
| Disconnected (unexpected) | Connection status shows red dot + "Disconnected", auto-reconnect attempt (up to 3 retries with 2s backoff), reconnecting spinner | Wait for reconnect, or end session |
| Error (unrecoverable) | Error toast with specific message (see error copy), fallback to text mode, error banner in avatar area | Switch to text mode, retry connection, or end session |

## Accessibility Contract

Inherited from Phase 1, plus Phase 8 additions:

| Requirement | Implementation |
|-------------|----------------|
| Color contrast | All status colors meet WCAG AA for their usage. Green/orange mic states use white foreground (7:1+ ratio). Floating transcript white-on-black-70% meets 4.5:1 minimum. |
| Focus indicators | Mic button receives visible focus ring (3px `--ring` color). Mode selector buttons are focusable via tab. Keyboard input toggle allows tabbing into ChatInput. |
| Keyboard navigation | Tab order: mode selector -> transcript area -> voice controls (mic, mute, keyboard toggle) -> side panel toggles. Space/Enter activates mic button. Escape closes end-session dialog. |
| Screen reader | `role="region"` and `aria-label` on avatar area, transcript area, voice controls. `aria-live="polite"` on transcript for new messages. `aria-live="assertive"` on connection status changes. Mic button `aria-label` changes per state (see Mic Button States table). |
| Touch targets | Mic button: 56px diameter (exceeds 44px minimum). Mute, keyboard, view toggles: 44px minimum. Mode selector buttons: 44px height. |
| Media autoplay | Avatar video uses `autoplay` and `playsinline` attributes. Audio playback initiated only after user interaction (mic button click satisfies browser autoplay policy). |
| Audio indicators | Waveform visualization provides visual feedback for audio that would otherwise be invisible, supporting deaf and hard-of-hearing users during voice sessions. |
