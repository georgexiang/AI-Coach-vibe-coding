<objective>
Wire voice session entry flow end-to-end: add Voice tab to scenario selection page (gated by feature flag), pass session mode through the full stack (frontend API -> backend API -> session service -> database), and expose voice_live_enabled feature flag from backend to frontend.

Purpose: Users currently have no way to start a voice session from the UI. The route exists (/user/training/voice) and all voice components are built (plans 01-03), but there is no entry point in the scenario selection page, the session creation API does not pass the mode parameter, and the voice_live_enabled feature flag is not exposed to the frontend.

Output: Working voice session entry flow from scenario selection, with feature toggle gating.
</objective>
