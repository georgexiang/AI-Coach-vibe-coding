    **CRITICAL (D-09, RESEARCH.md Pitfall 5): Transcript flush before end session.**
    The end-session handler MUST:
    1. Collect all pending transcript writes (track them via a `pendingFlushes` ref containing Promise[])
    2. `await Promise.all(pendingFlushesRef.current)` before calling the endSession mutation
    3. This prevents data loss where session ends before all transcripts are persisted

    Implementation pattern:
    ```typescript
    const pendingFlushesRef = useRef<Promise<void>[]>([]);

    // In transcript callback (when isFinal === true):
    const flushPromise = persistTranscriptMessage(sessionId, segment.role, segment.content);
    pendingFlushesRef.current.push(flushPromise);
    flushPromise.finally(() => {
      pendingFlushesRef.current = pendingFlushesRef.current.filter(p => p !== flushPromise);
    });

    // In end-session handler:
    const handleEndSession = async () => {
      // Flush all pending transcript writes first (D-09)
      await Promise.all(pendingFlushesRef.current);
      // Then disconnect voice
      await voiceLive.disconnect();
      avatarStream.disconnect();
      audioHandler.cleanup();
      // Then call endSession API
      endSessionMutation.mutate(sessionId);
    };
    ```
