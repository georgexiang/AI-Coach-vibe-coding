1. Create `backend/app/services/skill_validation_service.py`:

   Addresses review: L1 rules configurable via module-level constants (not hardcoded in logic), so they can be adjusted per locale/product later.

   **Module-level configurable rule constants:**
   ```python
   # Configurable L1 thresholds (addresses review: L1 rules may be overly rigid)
   MIN_SOP_STEPS = 3
   MIN_ASSESSMENT_ITEMS = 2
   MIN_SKILL_NAME_LENGTH = 2
   MIN_SKILL_DESCRIPTION_LENGTH = 10
   MIN_CONTENT_LENGTH = 50
   ERROR_PENALTY = 25
   WARNING_PENALTY = 10
   
   # Required stages (bilingual for zh-CN support)
   REQUIRED_STAGES = [
       ("Opening", "开场"),
       ("Product", "产品介绍", "产品"),
       ("Closing", "收尾", "总结"),
   ]
   ```

   **@dataclass ValidationIssue:**
   - `severity: str` -- "error" | "warning" | "info"
   - `dimension: str` -- Check dimension name
   - `message: str` -- Human-readable issue description
   - `suggestion: str` -- How to fix

   **@dataclass StructureCheckResult:**
   - `passed: bool`
   - `score: int` -- 0-100
   - `issues: list[ValidationIssue]`
   - `content_hash: str` -- SHA256 hash of skill.content at check time (for staleness detection)

   **def _compute_content_hash(content: str) -> str:**
   - Return hashlib.sha256(content.encode()).hexdigest()[:16]
   - Used to detect if content changed since last check

   **async def check_skill_structure(skill: Skill) -> StructureCheckResult:**
   Pure rule-based checks (no AI call -- instant, per D-10):

   a) **SOP Content Check:**
   - Count steps via regex: `r'^#{2,3}\s+(?:Step|步骤)\s*\d'` on content lines
   - If steps < MIN_SOP_STEPS: error "SOP has {n} steps, minimum is {MIN_SOP_STEPS}"
   - Check required stages by keyword matching (case-insensitive in content)
   - For each missing stage: warning "{stage} section may be missing"

   b) **Assessment Criteria Check:**
   - Look for "Assessment" or "考核" or "Rubric" sections
   - Count items (table rows or list items under assessment heading)
   - If items < MIN_ASSESSMENT_ITEMS: error "Assessment criteria has {n} items, minimum is {MIN_ASSESSMENT_ITEMS}"

   c) **Knowledge Points Check:**
   - Look for "Knowledge" or "知识" section
   - If no items found: warning "No key knowledge points found"

   d) **Basic Info Check:**
   - skill.name empty or < MIN_SKILL_NAME_LENGTH: error
   - skill.description empty or < MIN_SKILL_DESCRIPTION_LENGTH: warning
   - skill.content empty or < MIN_CONTENT_LENGTH: error

   e) **Score Calculation:**
   - Start at 100, subtract ERROR_PENALTY per error, WARNING_PENALTY per warning, min 0
   - passed = (error_count == 0)

   Return StructureCheckResult with content_hash = _compute_content_hash(skill.content)

   **def to_dict(result: StructureCheckResult) -> dict:**
   - Serialize including content_hash

1. Create `backend/app/services/skill_evaluation_service.py`:

   Addresses review: bind L2 evaluation to content hash so stale scores are detected on publish. Store per-dimension rationale. Cooldown between evaluations.

   **SKILL_EVALUATION_PROMPT** constant: system prompt for 6-dimension evaluation. Each dimension scored 0-100 with strengths, improvements, critical_issues, and rationale_text fields. Output JSON with overall_score, overall_verdict (PASS >= 70, NEEDS_REVIEW 50-69, FAIL < 50), dimensions array, summary, top_3_improvements. Include: "Evaluate the skill content objectively and provide specific rationale for each score."

   **Six dimensions (as named constants):**
   ```python
   EVALUATION_DIMENSIONS = [
       "sop_completeness",
       "assessment_coverage",
       "knowledge_accuracy",
       "difficulty_calibration",
       "conversation_logic",
       "executability",
   ]
   ```

   **@dataclass DimensionScore:**
   - name, score (int), verdict (str), strengths (list[str]), improvements (list[str]), critical_issues (list[str]), rationale (str)

   **@dataclass SkillEvaluationResult:**
   - overall_score (int), overall_verdict (str), dimensions (list[DimensionScore]), summary (str), top_improvements (list[str]), content_hash (str), evaluated_at (str -- ISO timestamp)

   **async def evaluate_skill_quality(db: AsyncSession, skill_id: str) -> SkillEvaluationResult:**
   - Load skill with selectinload(Skill.resources)
   - Compute content_hash via _compute_content_hash(skill.content) (imported from validation service)
   - Build prompt with skill data (truncate content to 50000 chars, reference material summaries to 5000 chars each)
   - Call Azure OpenAI (same pattern as scoring_engine.py): response_format={"type": "json_object"}, temperature=0.3
   - Validate response has required keys and all 6 dimensions
   - Store in skill: quality_score, quality_verdict, quality_details = json.dumps({**result, "content_hash": content_hash, "evaluated_at": datetime.utcnow().isoformat()})
   - db.flush()
   - Return result

   **def is_evaluation_stale(skill: Skill) -> bool:**
   - Parse quality_details JSON, extract content_hash
   - Compare with _compute_content_hash(skill.content)
   - Return True if they differ (content changed since last evaluation)

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Skill content -> LLM | Untrusted user content sent to Azure OpenAI for evaluation |
| LLM response -> DB | AI-generated scores stored as authoritative quality metrics |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-19-12 | Tampering | SOP content injection via crafted documents | mitigate | L1 structure check validates content before L2 evaluation. L2 prompt instructs objective evaluation. Content hash binds scores to specific content version. |
| T-19-13 | Elevation | Quality gate bypass via stale scores | mitigate | is_evaluation_stale() checks content_hash match before publish. Stale evaluations rejected with explicit error message. |
| T-19-14 | Elevation | Concurrent publish attempts | mitigate | publish_skill uses transactional gate: re-checks L1+L2+staleness within same DB transaction (db.flush). VALID_TRANSITIONS prevents double-publish. |
| T-19-15 | Denial of Service | L2 evaluation cost | accept | L2 is admin-triggered only, not automatic. Background task pattern prevents concurrent evals on same skill. |
</threat_model>
