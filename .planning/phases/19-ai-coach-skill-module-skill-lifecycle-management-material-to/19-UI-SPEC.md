## Interaction Contracts

### Skill Creation Flow

1. Admin clicks "+ Create Skill" on Skill Hub
2. Dialog appears: "Create from materials" OR "Import ZIP package"
3a. **From materials:** Navigate to `/admin/skills/new` with upload zone active
   - Admin drags files (PDF, DOCX, PPTX, TXT, MD)
   - Upload completes -> conversion starts (status: "processing")
   - Poll every 3 seconds until completed/failed
   - On complete: navigate to editor with Content tab active, SOP pre-filled
3b. **Import ZIP:** File picker opens, admin selects .zip file
   - Client-side extraction with jszip, preview structure in dialog
   - Confirm import -> POST to backend -> navigate to editor

### SOP Editing (Dual Mode)

- **Manual edit:** Direct Markdown textarea editing, live preview in right panel
- **AI feedback:** Textarea below SOP editor, "Apply Changes" button
  - Admin types modification instructions (e.g., "Add more objection handling for pricing concerns")
  - Button triggers async AI regeneration
  - On complete: SOP content updates in editor, diff highlighted temporarily (1.5s bg-primary/5 fade)

### Quality Gate Flow

1. Admin clicks "Request Quality Review" button (visible when status=draft/review)
2. L1 structure check runs immediately (< 1s, rule-based)
   - Result shown as banner: green "L1 PASS" or red "L1 FAIL: {reason}"
3. L2 AI assessment runs async (polling, ~10-30s)
   - Progress indicator during assessment
   - On complete: radar chart renders, dimension cards populate
4. When admin clicks "Publish":
   - If L1 FAIL: blocked, toast error "Please fix structure issues before publishing"
   - If L2 < 50: blocked, dialog shows low scores + "Cannot publish — quality score too low"
   - If L2 50-69: warning dialog with dimension breakdown + "Publish Anyway" / "Return to Edit"
   - If L2 >= 70: direct publish, toast success

### Lifecycle Transitions

| Current Status | Available Actions | Target Status |
|----------------|-------------------|---------------|
| draft | Edit, Request Review, Delete | review / (stays draft) |
| review | Edit (returns to draft), Publish, Delete | draft / published |
| published | Edit (creates new version as draft), Archive | draft (new version) / archived |
| archived | Restore | draft |
| failed | Retry Conversion, Delete | processing -> draft |

### File Tree Interaction

- Click folder: expand/collapse (chevron rotates)
- Click file: loads content in right preview panel
- SKILL.md: renders Markdown
- references/*: shows file info card (name, size, type) + download button
- scripts/*: renders code with font-mono, syntax highlighting via `<pre>` styling
- assets/*: shows file info + download button
- Upload button at tree top: opens file picker, adds to selected resource_type folder

### Scenario Skill Association

- In Scenario editor (existing page), add a "Skill" Select dropdown
- Shows only `published` status skills
- Selecting a skill shows skill name + quality score badge inline
- If associated skill is archived: show warning icon + tooltip "This skill is archived"

## State Specifications

### Loading States

| Context | Implementation |
|---------|---------------|
| Skill Hub cards loading | 6 Skeleton cards in grid (matching card dimensions) |
| Skill editor loading | Skeleton for tabs + content area |
| File tree loading | 4 Skeleton lines with indentation |
| Conversion in progress | Progress component (indeterminate) + "Converting..." text |
| L2 assessment running | Skeleton radar chart + "Evaluating quality..." text |

### Error States

| Context | Implementation |
|---------|---------------|
| Skill Hub fetch failed | EmptyState with error icon + "Failed to load skills" + Retry button |
| Conversion failed | Banner with destructive background + error message + "Retry" button |
| L2 assessment failed | Alert banner in Quality tab + "Try Again" button |
| File upload failed | Toast error via sonner + file stays in upload zone |
| ZIP import invalid | Dialog error message: "Invalid skill package format" |

### Empty States

| Context | Implementation |
|---------|---------------|
| Skill Hub (no skills) | EmptyState: BookOpen icon, "No skills yet", CTA "Create Skill" |
| Resources tab (no files) | EmptyState: FolderOpen icon, "No resources", CTA "Upload Files" |
| Quality tab (not evaluated) | Muted card: "Quality has not been evaluated. Request a review to see results." |
| File preview (no selection) | Centered muted text: "Select a file from the tree to preview" |
