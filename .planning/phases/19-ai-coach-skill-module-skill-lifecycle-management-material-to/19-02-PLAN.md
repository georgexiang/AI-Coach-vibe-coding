async def _call_openai_for_scoring(prompt: str) -> dict:
    settings = get_settings()
    client = AsyncOpenAI(api_key=settings.openai_api_key, ...)
    response = await client.chat.completions.create(
        model=settings.openai_model,
        messages=[{"role": "system", "content": prompt}],
        response_format={"type": "json_object"},
        temperature=0.3,
    )
    return json.loads(response.choices[0].message.content)

1. Create `backend/app/services/skill_text_extractor.py`:

   **extract_text_from_pdf(content: bytes) -> str:**
   - Use pdfplumber (better table support per D-05)
   - `import pdfplumber; import io`
   - Open `pdfplumber.open(io.BytesIO(content))`, iterate pages, extract_text()
   - Join pages with "\n\n", return text

   **extract_text_from_docx(content: bytes) -> str:**
   - Use python-docx: `from docx import Document; import io`
   - `doc = Document(io.BytesIO(content))`
   - Extract paragraphs: `[p.text for p in doc.paragraphs if p.text.strip()]`
   - Also extract table content: iterate `doc.tables`, for each row collect cell text
   - Join all with "\n"

   **extract_text_from_pptx(content: bytes) -> str:**
   - Use python-pptx: `from pptx import Presentation; import io`
   - `prs = Presentation(io.BytesIO(content))`
   - For each slide, collect all shape.text where hasattr(shape, "text") and shape.text.strip()
   - Separate slides with "\n\n---\n\n" (per RESEARCH.md pattern)

   **extract_text_from_text(content: bytes) -> str:**
   - Decode as utf-8 with errors='replace'
   - Return decoded string

   **extract_text(content: bytes, filename: str) -> str:**
   - Dispatch based on file extension (derive from filename, not MIME type per Phase 05 decision):
     - `.pdf` -> extract_text_from_pdf
     - `.docx` -> extract_text_from_docx
     - `.pptx` -> extract_text_from_pptx
     - `.txt`, `.md` -> extract_text_from_text
   - Return extracted text or empty string for unsupported types

   **convert_to_markdown(text: str, filename: str) -> str:**
   - For PDF/DOCX: text is already roughly paragraph-structured, add heading inference (lines < 80 chars followed by blank line -> `## heading`)
   - For PPTX: each slide section becomes a `## Slide N` heading with content below
   - For TXT/MD: return as-is (already Markdown or plain text)
   - This preserves structure for semantic chunking per D-06

2. Create `backend/app/services/skill_conversion_service.py`:

   **SOP_EXTRACTION_PROMPT** constant (use the exact prompt from RESEARCH.md code examples, which expects JSON output with sop_steps, assessment_criteria, key_knowledge_points, summary fields).

   **COACHING_PROTOCOL_TEMPLATE** constant: a Jinja2/f-string template that converts the extracted JSON into Markdown Coaching Protocol format:
   ```
   # Coaching Protocol: {skill_name}

   ## Overview
   {summary}

   ## SOP Steps

   ### Step {n}: {title}
   **Description:** {description}
   **Key Points:**
   - {point1}
   - {point2}
   **Common Objections:**
   - {objection1}
   **Assessment Criteria:**
   - {criterion}
   **Knowledge Points:**
   - {knowledge}
   **Suggested Duration:** {duration} minutes

   ## Assessment Rubric
   | Criterion | Weight | Passing Score |
   |-----------|--------|---------------|
   | {criterion} | {weight}% | {passing_score} |

   ## Key Knowledge Points
   ### {topic}
   {content}
   Importance: {importance}
   ```

   **semantic_chunk(text: str, max_tokens: int = 80000) -> list[str]:**
   - Split by heading boundaries (lines starting with `#`, `##`, `###`, or `---`)
   - If a section exceeds max_tokens (estimate: 1 token ~ 4 chars), split at paragraph boundaries
   - If still too large, split at sentence boundaries
   - Return list of text chunks, each under max_tokens
   - Per D-06: heading/section boundaries first, size as fallback

   **async def _call_sop_extraction(text_chunk: str, settings) -> dict:**
   - Use Azure OpenAI client (same pattern as scoring_engine.py)
   - Create from `openai.AsyncAzureOpenAI` if azure_openai_endpoint is configured, else `openai.AsyncOpenAI`
   - Call `client.chat.completions.create(model=..., messages=[{"role": "system", "content": SOP_EXTRACTION_PROMPT.format(document_text=text_chunk)}], response_format={"type": "json_object"}, temperature=0.3)`
   - Parse response JSON, validate it has required keys (sop_steps, assessment_criteria, key_knowledge_points)
   - Return parsed dict

   **def merge_extractions(parts: list[dict]) -> dict:**
   - Merge sop_steps from all parts (deduplicate by title similarity)
   - Merge assessment_criteria (deduplicate, normalize weights to sum=100)
   - Merge key_knowledge_points (deduplicate by topic)
   - Take first non-empty summary
   - Return merged dict

   **def format_coaching_protocol(extraction: dict, skill_name: str) -> str:**
   - Use COACHING_PROTOCOL_TEMPLATE to format the extracted data into Markdown body
   - Return formatted Markdown string

   **async def extract_resource_texts(db: AsyncSession, skill_id: str) -> None:**
   - Load all SkillResource where resource_type="reference" and extraction_status != "completed"
   - For each resource: read file from storage, call extract_text(), store result in resource.text_content, set extraction_status="completed"
   - On error: set extraction_status="failed"
   - db.flush()

   **async def start_conversion(db: AsyncSession, skill_id: str) -> Skill:**
   - Load skill, set conversion_status="processing", db.flush()
   - Try:
     1. Call extract_resource_texts(db, skill_id)
     2. Collect all text_content from reference resources
     3. Convert to markdown: convert_to_markdown() for each
     4. Concatenate all markdown
     5. Get chunk_limit from config (default 80000 per D-07): `chunk_limit = int(await _get_config_value(db, "skill_chunk_token_limit", "80000"))`
     6. Chunk: semantic_chunk(all_markdown, max_tokens=chunk_limit)
     7. Extract per chunk: extracted_parts = [await _call_sop_extraction(chunk, settings) for chunk in chunks]
     8. Merge: merged = merge_extractions(extracted_parts)
     9. Format: content = format_coaching_protocol(merged, skill.name)
     10. Update skill: skill.content = content, skill.conversion_status = "completed", skill.conversion_error = ""
   - Except Exception as e:
     - skill.conversion_status = "failed"
     - skill.conversion_error = str(e)
     - logger.error("Skill conversion failed for %s: %s", skill_id, e)
   - db.flush()
   - Return skill

   **_get_config_value helper:** Try to load from ServiceConfig table (service_name="skill_config", key), fallback to default. Use lazy import of config_service to avoid circular imports.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client -> Upload | Admin uploads arbitrary files — content could be malicious |
| Upload -> Text Extraction | Document parsing libraries process untrusted bytes |
| Extracted Text -> LLM | Large text sent to Azure OpenAI — token limit risk |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-19-05 | Tampering | File upload | mitigate | Extension whitelist (.pdf, .docx, .pptx, .txt, .md only), max 50MB per file, max 10 files per upload |
| T-19-06 | Tampering | PDF parsing via pdfplumber | accept | pdfplumber is well-maintained; decompression bombs are mitigated by page-level extraction. No arbitrary code execution risk. |
| T-19-07 | Denial of Service | Large document extraction | mitigate | Truncate combined text to 500K characters (~125K tokens) before sending to LLM. Log warning on truncation. Configurable chunk_limit (D-07). |
| T-19-08 | Elevation | Conversion endpoints | mitigate | All endpoints require require_role("admin") |
</threat_model>
