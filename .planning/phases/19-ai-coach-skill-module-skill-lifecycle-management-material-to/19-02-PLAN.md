   **extract_text_from_pdf(content: bytes) -> str:**
   - Use pdfplumber (better table support per D-05)
   - `import pdfplumber; import io`
   - Open `pdfplumber.open(io.BytesIO(content))`, iterate pages, extract_text()
   - Handle encrypted PDFs: catch Exception, return empty string with warning log
   - Join pages with "\n\n", return text

   **extract_text_from_docx(content: bytes) -> str:**
   - Use python-docx: `from docx import Document; import io`
   - `doc = Document(io.BytesIO(content))`
   - Extract paragraphs: `[p.text for p in doc.paragraphs if p.text.strip()]`
   - Also extract table content: iterate `doc.tables`, for each row collect cell text
   - Join all with "\n"

   **extract_text_from_pptx(content: bytes) -> str:**
   - Use python-pptx: `from pptx import Presentation; import io`
   - `prs = Presentation(io.BytesIO(content))`
   - For each slide: collect all shape.text where hasattr(shape, "text") and shape.text.strip()
   - Separate slides with "\n\n---\n\n"

   **extract_text_from_text(content: bytes) -> str:**
   - Decode as utf-8 with errors='replace'

   **extract_text(content: bytes, filename: str) -> str:**
   - Dispatch based on file extension (derive from filename, not MIME type per Phase 05 decision):
     - `.pdf` -> extract_text_from_pdf
     - `.docx` -> extract_text_from_docx
     - `.pptx` -> extract_text_from_pptx
     - `.txt`, `.md` -> extract_text_from_text
   - Wrap each extractor call in try/except: on failure log error and return empty string (addresses review: extraction edge cases)
   - Return extracted text

   **convert_to_markdown(text: str, filename: str) -> str:**
   - For PDF/DOCX: add heading inference (lines < 80 chars followed by blank line -> `## heading`)
   - For PPTX: each slide section becomes `## Slide N`
   - For TXT/MD: return as-is
   - Preserves structure for semantic chunking per D-06

1. Add conversion and regeneration endpoints to `backend/app/api/skills.py`.

   Addresses review: durable background processing. Instead of bare asyncio.create_task losing tasks on restart, use a wrapper that: (a) creates a new DB session, (b) re-checks conversion_job_id matches for idempotency, (c) commits independently, (d) handles errors with proper rollback.

   **Background task wrapper (define at module level):**
   ```python
   import asyncio
   import logging
   from app.database import AsyncSessionLocal

   logger = logging.getLogger(__name__)

   async def _run_durable_conversion(skill_id: str, job_id: str):
       """Durable background conversion with own session and idempotency check."""
       async with AsyncSessionLocal() as session:
           try:
               # Idempotency: verify job_id still matches
               skill = await session.get(Skill, skill_id)
               if not skill or skill.conversion_job_id != job_id:
                   logger.info("Conversion job %s superseded, skipping", job_id)
                   return
               await skill_conversion_service.start_conversion(session, skill_id)
               await session.commit()
           except Exception as e:
               await session.rollback()
               # Mark as failed in a new session to ensure the error is persisted
               async with AsyncSessionLocal() as err_session:
                   try:
                       skill = await err_session.get(Skill, skill_id)
                       if skill and skill.conversion_job_id == job_id:
                           skill.conversion_status = "failed"
                           skill.conversion_error = str(e)[:2000]
                           await err_session.commit()
                   except Exception:
                       logger.error("Failed to record conversion error for %s", skill_id)
               logger.error("Conversion task failed for %s: %s", skill_id, e)
   ```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client -> Upload | Admin uploads arbitrary files -- content could be malicious |
| Upload -> Text Extraction | Document parsing libraries process untrusted bytes |
| Extracted Text -> LLM | Large text sent to Azure OpenAI -- token limit risk |
| AI Feedback -> LLM | User feedback text could contain prompt injection |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-19-06 | Tampering | File upload | mitigate | Extension whitelist, max 50MB per file, max 10 files per upload (reuses Plan 01 validate_file_upload) |
| T-19-07 | Tampering | PDF/DOCX parsing | accept | pdfplumber/python-docx are well-maintained. Each extractor wrapped in try/except to contain failures. |
| T-19-08 | Denial of Service | Large document extraction | mitigate | Combined text truncated to 500K chars (~125K tokens) before LLM call. Configurable chunk_limit (D-07). |
| T-19-09 | Tampering | Prompt injection via documents | mitigate | SOP_EXTRACTION_PROMPT includes explicit instruction to evaluate objectively. JSON response_format constrains output. |
| T-19-10 | Tampering | AI feedback prompt injection | accept | Feedback text sent as user content. LLM output replaces SOP content which admin reviews before publishing. Admin-only operation. |
| T-19-11 | Denial of Service | Background conversion | mitigate | Idempotency via conversion_job_id prevents duplicate jobs. Status check prevents concurrent conversions. |
</threat_model>
