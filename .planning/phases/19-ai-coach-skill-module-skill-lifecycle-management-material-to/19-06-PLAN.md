1. Create `backend/app/services/skill_manager.py`:
   - Simplified adaptation of the reference `skill_manager.py`, focused on DB-based skill loading (not local files/git/clawhub):

   **@dataclass SkillContent:**
   - `name: str`, `description: str`, `content: str`, `token_estimate: int`

   **class SkillManager:**

   **@staticmethod
   def from_db_skill(skill: Skill) -> SkillContent:**
   - Return SkillContent(name=skill.name, description=skill.description, content=skill.content, token_estimate=len(skill.content) // 4)

   **@staticmethod
   def compose_instructions(base_instructions: str, skills: list[SkillContent], max_tokens: int = 4000) -> str:**
   - Follow the reference implementation pattern exactly:
     ```python
     parts = [base_instructions]
     total_tokens = len(base_instructions) // 4
     for skill in skills:
         header = f"== Skill: {skill.name} =="
         section = f"\n\n{header}\n{skill.description}\n\n{skill.content}"
         parts.append(section)
         total_tokens += skill.token_estimate
     if total_tokens > max_tokens:
         logger.warning("Total token estimate (%d) exceeds max_tokens (%d)", total_tokens, max_tokens)
     return "".join(parts)
     ```

   **async def load_skill_for_scenario(db: AsyncSession, scenario_id: str) -> SkillContent | None:**
   - Query Scenario by id, get skill_id
   - If no skill_id: return None
   - Load Skill by skill_id
   - If skill not found or status not in ("published", "archived"): return None
   - Return SkillContent.from_db_skill(skill)

   **def run_skill_script(script_content: str, timeout: int = 30) -> str:**
   - Per D-25: simple subprocess.run() + timeout
   ```python
   import subprocess
   result = subprocess.run(
       ["python3", "-c", script_content],
       capture_output=True, text=True, timeout=timeout
   )
   if result.returncode != 0:
       raise RuntimeError(f"Script failed: {result.stderr}")
   return result.stdout
   ```
   - Only used for lightweight validation scripts per D-24

   **async def read_skill_resource(db: AsyncSession, skill_id: str, resource_type: str, filename: str) -> str:**
   - Per D-26: load specific resource content on demand
   - Query SkillResource by skill_id + resource_type + filename
   - Return text_content or read from storage
