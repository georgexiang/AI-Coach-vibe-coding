1. Create `backend/app/services/skill_service.py` with these async functions:

   **create_skill(db: AsyncSession, data: SkillCreate, user_id: str) -> Skill:**
   - Create Skill record with data fields, created_by=user_id, status="draft"
   - Create initial SkillVersion (version_number=1, content=data.content, metadata_json=data.metadata_json)
   - db.flush()
   - Return skill (use selectinload for resources and versions relationships)

   **get_skills(db, page=1, page_size=20, status=None, product=None, search=None) -> tuple[list[Skill], int]:**
   - Query skills with optional filters: status exact match, product exact match, search (ILIKE on name/description)
   - Order by created_at desc
   - Return (items, total) for pagination

   **get_published_skills(db, page=1, page_size=20, search=None) -> tuple[list[Skill], int]:**
   - Same as get_skills but hardcoded status="published" filter

   **get_skill(db: AsyncSession, skill_id: str) -> Skill:**
   - Load Skill by id with selectinload(Skill.versions) and selectinload(Skill.resources)
   - Raise not_found("Skill not found") if not exists

   **update_skill(db: AsyncSession, skill_id: str, data: SkillUpdate) -> Skill:**
   - Load skill, apply non-None fields from data
   - If status change requested, validate lifecycle transition:
     - draft -> review (allowed)
     - review -> draft (allowed — edit returns to draft)
     - review -> published (allowed — but actual publish goes through publish_skill function)
     - published -> archived (allowed)
     - archived -> draft (allowed — restore)
     - failed -> draft (allowed — after retry)
     - All other transitions: raise bad_request("Invalid status transition from {current} to {target}")
   - db.flush()
   - Return updated skill

   **delete_skill(db: AsyncSession, skill_id: str) -> None:**
   - Load skill. Only allow deletion if status in ("draft", "failed").
   - Raise bad_request("Cannot delete a published or archived skill") otherwise.
   - Delete associated resources' files from storage.
   - db.delete(skill), db.flush()

   **publish_skill(db: AsyncSession, skill_id: str) -> Skill:**
   - Load skill. Check: structure_check_passed must be True, quality_score must be not None.
   - If structure_check_passed is not True: raise bad_request("L1 structure check must pass before publishing")
   - If quality_score is not None and quality_score < 50: raise bad_request("Quality score must be at least 50 to publish")
   - Set status="published"
   - Create new SkillVersion snapshot (version_number=current_version, content=skill.content, metadata_json=skill.metadata_json, is_active=True)
   - Mark all previous versions as is_active=False
   - db.flush()

   **create_new_version(db: AsyncSession, skill_id: str) -> Skill:**
   - Load published skill. Create a copy as new draft:
   - Increment current_version
   - Set status="draft"
   - Reset quality fields (structure_check_passed=None, quality_score=None, quality_verdict=None, quality_details="{}", structure_check_details="{}")
   - db.flush()

   **archive_skill(db: AsyncSession, skill_id: str) -> Skill:**
   - Load skill, verify status="published"
   - Set status="archived"
   - db.flush()

   **restore_skill(db: AsyncSession, skill_id: str) -> Skill:**
   - Load skill, verify status="archived"
   - Set status="draft"
   - db.flush()

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client -> API | Admin uploads files and creates/edits skills — untrusted input |
| API -> Storage | Files stored to local filesystem — path injection risk |
| API -> Database | Skill content stored — injection through metadata fields |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-19-01 | Tampering | POST /skills resource upload | mitigate | Validate file extension against whitelist (pdf, docx, pptx, txt, md); reject others. Enforce max file size 50MB per file. |
| T-19-02 | Tampering | Resource storage path | mitigate | Sanitize filename with `secure_filename()` from werkzeug or equivalent; reject filenames containing `..` or absolute paths. Storage path always constructed as `skills/{skill_id}/{type}s/{safe_filename}`. |
| T-19-03 | Elevation | All /skills/* endpoints | mitigate | All routes use `require_role("admin")` dependency — non-admin users get 403. |
| T-19-04 | Information Disclosure | GET /resources/{id}/download | mitigate | Use FileResponse with content_disposition, never expose storage_path to client. Return download URL pattern `/api/v1/skills/{id}/resources/{rid}/download`. |
</threat_model>
