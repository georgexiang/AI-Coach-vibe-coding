**Session State Machine (addresses review concern: "define a strict session state machine"):**
```typescript
type SessionState = "idle" | "connecting" | "connected" | "error" | "stopping";
```
Use `useState<SessionState>("idle")` instead of separate boolean flags. This makes the state transitions deterministic:
- `idle` -> `connecting` (Start clicked)
- `connecting` -> `connected` (all hooks initialized) OR `error` (any failure)
- `connected` -> `stopping` (Stop clicked or unmount)
- `stopping` -> `idle` (cleanup complete)
- `error` -> `idle` (user acknowledges or retries)

**TRANSCRIPT BUFFER CAP (addresses review concern: "Transcript buffer growth (LOW)"):**
Cap `transcripts` array at 100 segments. In the `onTranscript` callback, after appending a new segment, if `prev.length >= 100`, remove the oldest non-final segment to keep memory bounded:
```typescript
const MAX_TRANSCRIPTS = 100;
// Inside onTranscript callback:
if (next.length > MAX_TRANSCRIPTS) {
  next = next.slice(next.length - MAX_TRANSCRIPTS);
}
```
