    # Pre-Demo Smoke Test Checklist

    > Manual verification checklist for BeiGene AI Coach Platform demo preparation.
    > Run through this checklist before every customer demo to ensure all services are operational.
    >
    > **Prerequisites:**
    > - Backend running: `cd backend && uvicorn app.main:app --port 8000`
    > - Frontend running: `cd frontend && npm run dev`
    > - Azure credentials configured via Admin Azure Config page
    > - Voice SDK installed: `cd backend && pip install -e ".[voice]"` (if testing speech features)

    ---

    ## 1. Azure Service Health

    Navigate to **Admin > Azure Config** (`/admin/azure-config`).

    - [ ] **Azure OpenAI:** Expand card > click "Test Connection" > green success toast within 15 seconds
    - [ ] **Azure Speech (STT):** Expand card > click "Test Connection" > green success toast
    - [ ] **Azure Speech (TTS):** Expand card > click "Test Connection" > green success toast
    - [ ] **Azure Voice Live:** Expand card > verify region is `eastus2` or `swedencentral` > click "Test Connection" > success toast
    - [ ] **Azure AI Avatar:** Expand card > click "Test Connection" > "Configuration valid" message (avatar uses WebRTC, no HTTP ping)

    ---

    ## 2. Text Mode Demo

    Navigate to **Scenario Selection** (`/user/scenario-selection`), select a scenario, start session.

    - [ ] Scenario selection page loads with available scenarios
    - [ ] Click "Start Session" opens the coaching chat interface
    - [ ] Send first message: AI HCP responds within 3 seconds (D-11)
    - [ ] Send 3+ messages: AI responds in character (personality matches HCP profile)
    - [ ] Real-time coaching hints appear in side panel
    - [ ] Key message delivery checklist updates as messages are sent
    - [ ] Click "End Session" > confirmation dialog appears
    - [ ] Confirm end > scoring report renders with all dimensions
    - [ ] Scoring report shows: overall score, per-dimension scores, strengths, weaknesses, improvement suggestions
    - [ ] Conversation quotes appear in the feedback sections

    ---

    ## 3. Voice Mode Demo

    Start a voice-enabled session (if Voice Live is configured).

    - [ ] Voice session page loads without errors
    - [ ] Connection status transitions: "Connecting..." > "Connected" (green indicator)
    - [ ] Mic button appears and responds to click (starts/stops recording)
    - [ ] Speak into mic: transcript appears in real-time on screen
    - [ ] AI HCP responds with spoken voice (audio plays through speaker)
    - [ ] Waveform visualization animates during AI speech
    - [ ] End session > scoring report generates correctly
    - [ ] Scoring quality matches text mode (same dimensions, comparable depth)

    ---

    ## 4. Avatar Mode Demo

    Start a session with Avatar enabled (if Avatar service is configured).

    - [ ] Avatar video renders in the session UI (no black screen, no freeze)
    - [ ] Lip-sync matches AI speech output (visual alignment, D-12)
    - [ ] Avatar gestures and expressions change during conversation
    - [ ] No video glitches, stuttering, or frame drops during 30-second observation
    - [ ] Audio and avatar are synchronized (no perceptible delay)
    - [ ] End session > scoring report generates correctly

    ---

    ## 5. Fallback Chain Verification (D-13)

    Test graceful degradation when services are unavailable.

    ### 5a. Avatar > Voice-Only Fallback
    - [ ] Disable Avatar in Admin Azure Config (remove key or deactivate)
    - [ ] Start session that would normally use avatar
    - [ ] Verify: waveform visualization appears instead of avatar video
    - [ ] Verify: voice interaction still works (STT input, TTS output)
    - [ ] Verify: toast notification or status message indicates avatar unavailable

    ### 5b. Voice > Text-Only Fallback
    - [ ] Disable Voice Live and Speech in Admin Azure Config
    - [ ] Start a session
    - [ ] Verify: text chat interface appears (ChatInput with textarea)
    - [ ] Verify: no voice controls visible (no mic button, no waveform)
    - [ ] Verify: text coaching works normally with mock or Azure OpenAI adapter

    ### 5c. Full Fallback (No Azure services)
    - [ ] Disable all Azure services in Admin Azure Config
    - [ ] Start a session
    - [ ] Verify: mock adapter handles conversation (responses prefixed with "[Mock HCP Response]" or similar)
    - [ ] Verify: scoring report still generates (using mock scoring)
    - [ ] Verify: no error screens, no broken UI, no unhandled exceptions in console

    ---

    ## 6. Scoring Parity (D-14)

    Compare scoring reports across session modes.

    - [ ] Text session scoring report has: overall score, 5-6 dimension scores, strengths, weaknesses, suggestions
    - [ ] Voice session scoring report has: same dimensions and structure as text
    - [ ] Avatar session scoring report has: same dimensions and structure as text
    - [ ] Scores appear reasonable (not all zeros, not all 100s)
    - [ ] Conversation quotes reference actual transcript content

    ---

    ## 7. Performance Verification (D-11)

    - [ ] **Text mode:** First AI response appears within 3 seconds of sending message
    - [ ] **Text mode:** Full response completes within 15 seconds
    - [ ] **Voice mode:** Speech is recognized within 5 seconds of speaking
    - [ ] **Voice mode:** AI voice response begins within 3 seconds of transcript display
    - [ ] **Admin config:** Test Connection completes within 15 seconds for each service
    - [ ] **Page loads:** All pages load within 3 seconds (no blank screens, no spinners lasting >5s)

    ---

    ## 8. Cross-Browser Quick Check

    - [ ] Open the app in Chrome (primary)
    - [ ] Open the app in Edge (secondary, likely used in BeiGene)
    - [ ] Verify login, dashboard, and scenario selection render correctly in both browsers

    ---

    ## Quick Fix Reference

    | Issue | Fix |
    |-------|-----|
    | "Connection failed" on all services | Check backend .env has correct Azure credentials |
    | Mock responses instead of real AI | Verify Azure OpenAI is configured and "Test Connection" passes |
    | Voice session stuck on "Connecting" | Check Voice Live region is `eastus2` or `swedencentral` |
    | Avatar black screen | Verify WebRTC is not blocked by firewall/proxy |
    | Scoring report empty | Check session was properly ended (not just closed browser) |
    | STT not recognizing speech | Check microphone permissions in browser |

    ---

    *Last updated: Phase 09 - Integration Testing*
