1. Create `backend/app/services/conference_service.py`:
   - `async def create_conference_session(db: AsyncSession, scenario_id: str, user_id: str) -> CoachingSession`:
     - Verify scenario.mode == "conference" (raise 409 NOT_CONFERENCE_SCENARIO if not)
     - Load ConferenceAudienceHcp records with selectinload(hcp_profile) for the scenario
     - Require >= 2 audience HCPs (raise 409 INSUFFICIENT_AUDIENCE if not)
     - Build audience_config JSON from HCP profiles (hcp_profile_id, name, specialty, personality_type, role, voice_id)
     - Build key_messages_status from scenario.key_messages
     - Create CoachingSession with session_type="conference", sub_state="presenting"
     - Return session after db.flush()
   - `async def generate_hcp_questions(db: AsyncSession, session: CoachingSession, mr_text: str) -> list`:
     - Parse audience_config JSON
     - For each HCP (sequential, NOT parallel -- per RESEARCH.md Pitfall 4):
       - Build conference audience prompt via prompt_builder.build_conference_audience_prompt()
       - Call LLM adapter via ServiceRegistry.get("llm") to generate a question
       - Assign relevance_score based on keyword matching (mock) or LLM output
       - Add to turn_manager queue if question generated
     - Return list of newly queued questions
   - `async def handle_respond(db: AsyncSession, session: CoachingSession, hcp_id: str, mr_response: str) -> AsyncIterator[dict]`:
     - Activate question in turn_manager
     - Save MR response as SessionMessage with speaker_id=None, role="user"
     - Build follow-up prompt for target HCP
     - Stream HCP follow-up via LLM adapter as SSE events with speaker attribution
     - Save HCP response as SessionMessage with speaker_id=hcp_id, speaker_name=hcp_name
     - Mark question answered in turn_manager
   - `async def transition_sub_state(db: AsyncSession, session_id: str, new_state: str) -> None`:
     - Update session.sub_state, flush
   - `async def end_conference_session(db: AsyncSession, session_id: str) -> CoachingSession`:
     - Set status="completed", cleanup turn_manager
     - Trigger scoring via existing scoring_service.score_session()

2. Extend `backend/app/services/prompt_builder.py` with:
   - `def build_conference_audience_prompt(hcp_profile, scenario, presentation_topic, conversation_history, other_hcp_questions) -> str`:
     - System prompt instructs HCP to act as conference audience member
     - Includes HCP personality, specialty, and knowledge from profile
     - Includes conversation history and other HCPs' prior questions to avoid duplication
     - Asks for a relevant question about the MR's presentation (or empty string if no question)
   - `def build_conference_scoring_prompt(scenario, messages, audience_config) -> str`:
     - Adapts existing scoring prompt for conference: "key_message" evaluates presentation completeness, "objection_handling" evaluates Q&A handling, "communication" evaluates presentation delivery

3. Create `backend/app/api/conference.py` router:
   - `router = APIRouter(prefix="/conference", tags=["conference"])`
   - `POST /sessions` -- create conference session (requires auth)
   - `GET /sessions/{session_id}` -- get conference session details
   - `POST /sessions/{session_id}/stream` -- SSE endpoint for conference interaction
     - Accepts ConferenceMessageSend body (action: "present" or "respond", message, target_hcp_id)
     - Returns EventSourceResponse with conference events
     - Includes heartbeat every 15 seconds via asyncio background task
     - Events: text, speaker_text, queue_update, turn_change, sub_state, transcription, hint, key_messages, heartbeat, done, error
   - `PATCH /sessions/{session_id}/sub-state` -- transition presenting/qa
   - `POST /sessions/{session_id}/end` -- end session and trigger scoring
   - `GET /scenarios/{scenario_id}/audience` -- get audience HCPs for scenario
   - `PUT /scenarios/{scenario_id}/audience` -- set audience HCPs (admin only, accepts list of AudienceHcpCreate)
   - Static routes before parameterized routes per CLAUDE.md Gotcha #3

4. Register conference router in `backend/app/main.py`:
   - `from app.api.conference import router as conference_router`
   - `app.include_router(conference_router, prefix=settings.api_prefix)`
   - Place after existing router registrations
