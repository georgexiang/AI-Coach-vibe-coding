## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px (text-sm) | 400 (normal) | 1.5 |
| Label | 14px (text-sm) | 400 (normal) | 1.5 |
| Heading | 16px (text-base) | 600 (semibold) | 1.5 |
| Display | 24px (text-2xl) | 600 (semibold) | 1.5 |

Two weights only: 400 (normal) for body text and labels, 600 (semibold) for headings and display. Labels are differentiated from body text via color (using `text-muted-foreground` for secondary labels) or size context, not weight. Card titles use Heading (16px/600) per established `CardTitle` pattern with `text-base font-semibold`.

## Interaction Contracts

### I-01: HCP Editor Tab Navigation (D-05)

**Trigger:** Admin clicks a tab trigger (Profile / Voice & Avatar / Agent).
**Behavior:** Radix Tabs switches visible content instantly. All three TabsContent panels remain mounted in DOM (Radix default `forceMount` behavior). Form state from react-hook-form persists across tab switches because a single form instance wraps all tabs.
**Visual:** Active tab trigger shows `bg-background` with shadow (existing TabsTrigger style). Inactive triggers show `text-muted-foreground`.
**Constraint:** Tab switching must NOT trigger form validation. Validation only runs on Save button click.

### I-02: Voice Name Selection (D-01, D-04)

**Trigger:** Admin interacts with voice name field.
**Behavior:** When "Custom voice" switch is OFF (default), show a Select dropdown with preset voice options. When toggled ON, show a text Input for free-form voice name entry. Default value: "en-US-AvaNeural" per D-04.
**Visual:** Select dropdown with grouped options (English voices, Chinese voices). Each option shows voice label. Input appears below switch when custom is enabled, with placeholder "e.g., en-US-Ava:DragonHDLatestNeural".

### I-03: Avatar Character + Style Selection (D-03)

**Trigger:** Admin selects avatar character.
**Behavior:** Two linked Select dropdowns. Character dropdown shows available video avatar characters (harry, jeff, lisa, lori, max, meg). When character changes, style dropdown updates to show only valid styles for that character. When "Custom avatar" switch is ON, character becomes a text Input. Default: character "lori", style "casual" per D-04.
**Visual:** Side-by-side Select dropdowns (grid-cols-2 gap-4). Character label left, style label right. Custom toggle Switch below.

### I-04: Conversation Parameters (D-01)

**Trigger:** Admin adjusts conversation parameters in Voice & Avatar tab.
**Behavior:** Temperature uses Slider (0.0-1.0, step 0.1, default 0.9). Turn detection uses Select with 4 options (server_vad default). Noise suppression, echo cancellation, EOU detection each use Switch (all default OFF). Recognition language uses Select with options including "Auto Detect" (default).
**Visual:** Stacked form fields within a Card. Each row: FormLabel left-aligned, control right-aligned or below. Switches render inline with label (flex row, justify-between). Slider shows current value as text to the right.

### I-05: Agent Instructions Override (D-02)

**Trigger:** Admin views Agent tab.
**Behavior:** Auto-generated instructions text (derived from HCP personality fields) displayed in a read-only Textarea with `disabled` attribute and muted background. Below it, an editable Textarea labeled "Override Instructions" allows admin to write custom instructions. If override is non-empty, it replaces auto-generated text when syncing to AI Foundry.
**Visual:** Two Textareas stacked vertically. Top one: `bg-muted/50` disabled look, 6 rows. Bottom one: standard input style, 6 rows, placeholder "Leave empty to use auto-generated instructions".

### I-06: HCP Table Voice+Avatar Column (D-06, D-07)

**Trigger:** Table renders.
**Behavior:** New column after "Agent Status". Shows voice name and avatar character+style as two inline Badge elements. If HCP has no voice configuration (all defaults or empty), shows "Not configured" in muted text.
**Visual:** Two `Badge variant="outline"` side by side with 4px gap. First badge: voice name text (e.g., "Ava"). Second badge: character-style combined text (e.g., "Lori-casual"). Both badges use `text-xs`. When not configured: plain `text-xs text-muted-foreground` text.

### I-07: Auto-Mode Resolution (D-10)

**Trigger:** Voice session starts, token broker response received.
**Behavior:** `resolveMode(tokenData)` determines the best available mode:
1. If `avatar_enabled && agent_id` -> `digital_human_realtime_agent`
2. If `avatar_enabled` -> `digital_human_realtime_model`
3. If `agent_id` -> `voice_realtime_agent`
4. Otherwise -> `voice_realtime_model`
MR never sees a mode picker. Mode is auto-selected.
**Visual:** No ModeSelector rendered. ModeStatusIndicator in header shows resolved mode.

### I-08: Fallback Chain (D-11, D-12)

**Trigger:** Avatar connection fails during session, or voice connection degrades.
**Behavior:** Three-level fallback: Digital Human Realtime Agent -> Voice-only Realtime -> Text mode. Each fallback triggers:
1. Toast notification via sonner: "Avatar unavailable, switching to voice mode" or "Voice unavailable, switching to text mode"
2. ModeStatusIndicator updates to show new active mode with amber dot (degraded) instead of green dot (optimal)
**Visual:** Toast uses `toast.warning()` (amber styling). ModeStatusIndicator badge text updates dynamically. Amber 8px dot replaces green dot on degradation.

### I-09: Mode Status Indicator (D-12)

**Trigger:** Always visible during voice session.
**Behavior:** Renders in VoiceSessionHeader center area. Shows:
- Mode label text from i18n (e.g., "Digital Human Agent", "Voice Only", "Text")
- Colored dot: green (8px circle `bg-strength`) when at optimal mode, amber (`bg-weakness`) when degraded/fell back, red (`bg-destructive`) when disconnected
**Visual:** `Badge variant="outline"` with prepended colored dot. Total height 28px. Text `text-xs font-semibold`. Gap between dot and text: 8px (`gap-2`).

### I-10: Token Broker Per-HCP Wiring (D-08)

**Trigger:** MR starts a voice session for a scenario with an HCP.
**Behavior:** Frontend passes `hcp_profile_id` to token broker API. Backend reads HCP voice/avatar settings and returns them in `VoiceLiveTokenResponse`. `useVoiceLive` hook and `useAvatarStream` hook consume per-HCP settings (voice name, avatar character/style, temperature, turn detection, etc.) from token response to configure WebSocket session and WebRTC avatar.
**Visual:** No visible UI change. Settings are applied automatically. MR sees the correct avatar character and hears the correct voice.

### I-11: End Session with Flush (existing, unchanged)

**Trigger:** MR clicks End Session button.
**Behavior:** Dialog confirmation. On confirm: flush pending transcripts, disconnect voice/avatar, call endSession API, navigate to scoring page.
**Visual:** Existing Dialog pattern from Phase 08. No changes.

## State Management

| State | Type | Location | Purpose |
|-------|------|----------|---------|
| HCP form values (all tabs) | react-hook-form `useForm` | `hcp-profile-editor.tsx` | Single form instance across Profile / Voice & Avatar / Agent tabs. Prevents data loss on tab switch. |
| Active tab | Radix Tabs internal | `hcp-profile-editor.tsx` | `defaultValue="profile"`. Uncontrolled. |
| Current session mode | `useState<SessionMode>` | `voice-session.tsx` | Auto-resolved from token broker, updated on fallback. |

| Token broker response | TanStack Query mutation | `use-voice-token.ts` | Extended to pass `hcp_profile_id`. Returns all per-HCP voice/avatar/conversation params. |
