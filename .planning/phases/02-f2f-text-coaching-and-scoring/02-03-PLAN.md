**backend/app/api/hcp_profiles.py** — Admin-only CRUD router:

```python
router = APIRouter(prefix="/hcp-profiles", tags=["hcp-profiles"])
```

All endpoints require `user: User = Depends(require_role("admin"))` for admin-only access.

Endpoints:
- `POST /` — Create HCP profile. Request: `HcpProfileCreate`. Response: `HcpProfileResponse`, status 201.
- `GET /` — List HCP profiles. Query params: `page: int = 1, page_size: int = 20, search: str | None = None, is_active: bool | None = None`. Response: paginated using `PaginatedResponse`.
- `GET /{profile_id}` — Get single profile. Response: `HcpProfileResponse`.
- `PUT /{profile_id}` — Update profile. Request: `HcpProfileUpdate`. Response: `HcpProfileResponse`.
- `DELETE /{profile_id}` — Delete profile. Response: 204 No Content.

For list responses, the HcpProfileResponse needs to handle JSON text -> list conversion. Add a `@field_validator` or `@model_validator` on HcpProfileResponse that parses JSON string fields (expertise_areas, objections, probe_topics) into Python lists. Example:
```python
@field_validator("expertise_areas", "objections", "probe_topics", mode="before")
@classmethod
def parse_json_list(cls, v):
    if isinstance(v, str):
        return json.loads(v)
    return v
```

**backend/app/api/scenarios.py** — Admin-only CRUD router:

```python
router = APIRouter(prefix="/scenarios", tags=["scenarios"])
```

Endpoints (all admin-only):
- `POST /` — Create scenario. Request: `ScenarioCreate`. Response: `ScenarioResponse`, status 201.
- `GET /` — List scenarios. Query params: `page, page_size, status, mode, search`. Response: paginated.
- `GET /active` — List active scenarios for user selection (this endpoint is accessible by authenticated users, not admin-only). Uses `Depends(get_current_user)`.
- `GET /{scenario_id}` — Get single scenario with eager-loaded HCP profile. Response: `ScenarioResponse`.
- `PUT /{scenario_id}` — Update scenario. Request: `ScenarioUpdate`.
- `DELETE /{scenario_id}` — Delete scenario. 204.
- `POST /{scenario_id}/clone` — Clone scenario. Response: `ScenarioResponse`, status 201.

IMPORTANT: Put `GET /active` BEFORE `GET /{scenario_id}` (Gotcha #3 — static before parameterized routes).

Add `@field_validator` on ScenarioResponse for `key_messages` JSON text -> list conversion (same pattern as HCP profiles).
