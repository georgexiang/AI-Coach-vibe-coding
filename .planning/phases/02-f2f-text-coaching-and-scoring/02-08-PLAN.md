**backend/app/api/azure_config.py** — Azure service configuration endpoint for PLAT-03:

```python
router = APIRouter(prefix="/azure-config", tags=["azure-config"])
```

Admin-only endpoints:
- `GET /services` — List all Azure services with their current config status (connected/not configured). Returns service name, status (based on whether endpoint and key are set in settings), and masked key (last 4 chars).
- `PUT /services/{service_name}` — Update service config (endpoint, api_key, model, region). Accepts JSON body. For MVP, store in settings or a config table. For simplest approach: write to `.env` file or a separate config store.
- `POST /services/{service_name}/test` — Test connection to a specific Azure service. For Azure OpenAI: make a simple API call. For mock/MVP: return success if endpoint and key are non-empty strings matching expected format (starts with `https://`).

For MVP implementation, the simplest approach:
- Read current config from `get_settings()` (env vars)
- Test endpoint does a basic HTTP health check or OpenAI list-models call
- Update endpoint stores in a JSON config file (since env vars can't be hot-reloaded easily)
- OR: just validate the format and return success for demo purposes
