<objective>
Build the Voice Live Management admin page with chain status visualization, summary statistics, batch re-sync, route registration, and sidebar navigation -- the culmination of Phase 13 that mirrors the AI Foundry portal's Voice Live workflow.

Purpose: Gives admin a single overview page to see the full HCP -> Agent -> Voice Live -> Speech/Avatar pipeline status across all profiles, with ability to re-sync and navigate to edit individual HCPs. This matches the AI Foundry portal's dedicated Voice Live section.

Output: VoiceLiveChainCard component, Voice Live Management page, batch re-sync hook, admin route, sidebar nav item, backend tests, build verification.

**Endpoint note:** The UI-SPEC (I-05) references a `POST /api/v1/voice-live/batch-resync` endpoint. This plan deliberately reuses the existing `POST /api/v1/hcp-profiles/batch-sync` endpoint instead of creating a new one. The existing endpoint already performs exactly the same operation (batch agent sync for all HCP profiles). There is no need to duplicate this endpoint under a different URL. The frontend calls `batchSyncAgents()` from `frontend/src/api/hcp-profiles.ts`, which invokes `POST /hcp-profiles/batch-sync`.
</objective>

```typescript
import { useMutation, useQueryClient } from "@tanstack/react-query";
import { batchSyncAgents } from "@/api/hcp-profiles";

export function useBatchResyncAgents() {
  const queryClient = useQueryClient();
  return useMutation({
    mutationFn: () => batchSyncAgents(),
    onSuccess: () => {
      queryClient.invalidateQueries({ queryKey: ["hcp-profiles"] });
    },
  });
}
```
