<objective>
Add per-HCP Voice Live model selection to the backend: new database column, schema extensions, token broker per-HCP model resolution, supported model list constant, and new API endpoint for fetching the model list.

Purpose: Enables admin to select which generative AI model (GPT-4o, GPT-4.1, GPT-5, etc.) each HCP uses for Voice Live sessions, matching the AI Foundry portal workflow where each agent can have a different model tier.

Output: Alembic migration, extended ORM/schemas, token broker with per-HCP model, model list endpoint, backend tests.

**Deferred:** The `azure-ai-projects` SDK upgrade from `1.0.0b12` to `2.0.1` (identified in RESEARCH.md) is NOT included in this phase. Rationale: Phase 13 adds no new Azure API calls -- the existing SDK version supports all current agent sync and metadata operations. The SDK upgrade carries regression risk for the agent sync pipeline (Phase 11) and should be done in a dedicated future phase with focused testing. The model selection feature works entirely through the existing token broker and WebSocket connection parameters, which do not depend on the azure-ai-projects SDK.
</objective>

```python
@router.get("/models", response_model=VoiceLiveModelsResponse, status_code=200)
async def get_voice_live_models(
    user: User = Depends(get_current_user),
) -> VoiceLiveModelsResponse:
    """Return the list of supported Voice Live generative AI models."""
    from app.services.voice_live_models import VOICE_LIVE_MODELS
    from app.schemas.voice_live import VoiceLiveModelInfo
    models = [
        VoiceLiveModelInfo(
            id=model_id,
            label=info["label"],
            tier=info["tier"],
            description=info["description"],
        )
        for model_id, info in VOICE_LIVE_MODELS.items()
    ]
    return VoiceLiveModelsResponse(models=models)
```
